# toolbox 模块说明

## 模块概述
`toolbox` 模块聚合睡眠支持、声音工具、专注训练、静心减压、趣味测试和随机决策等子工具。

## 四件套落地状态
- 状态独立: 进行中（子工具各自维护局部状态，仍有部分页面共享 `AppState`）。
- 仓库独立: 部分完成（ambient 路径已仓库化，子工具数据边界仍在推进）。
- 注册驱动: 已落地（入口分组与子模块均由 `ModuleRegistry + module toggle` 驱动）。
- 启停守卫: 已落地（页面访问守卫 + 路由跳转守卫统一接入，禁用后阻断导航）。

## 当前依赖与入口
- 入口: `AppHomeTab.toolbox` -> `ToolboxPage`
- 子模块入口: `toolbox_page_content.dart` 的 `ToolboxEntryData.moduleId`
- 路由守卫: `ui/module/module_access.dart` 的 `pushModuleRoute`

## 人类测试
- 入口: `HumanTestsToolPage`
- 页面编排: `lib/src/ui/pages/toolbox_human_tests.dart`
- 展示与状态拆分:
  - `toolbox_human_tests_shared.dart`: hub 卡片、测试舞台和共享指标组件。
  - `toolbox_human_tests_action.dart`: 反应测试、瞄准测试、手速测试、时间感知测试、手眼协调测试。
  - `toolbox_human_tests_memory.dart`: 数字记忆、黑猩猩测试、视觉记忆、序列记忆。
  - `toolbox_human_tests_visual.dart`: 色觉测试、动态视力测试。
  - `toolbox_human_tests_cognition.dart`: 打字测试、斯特鲁普、词汇记忆、运气测试、计算能力测试、持续注意力测试。
- 当前能力:
  - 按 Human Benchmark 类条目组织 17 个本地趣味测试，覆盖反应、短时记忆、视觉辨识、手眼协调、打字、计算和持续注意力。
  - 测试结果仅在当前页面即时展示，不保存到 AppState、主数据库或学习记录。
  - 当前实现只参考网页条目和规则结构，未复制外站源码或样式。
- 后续扩展预留:
  - 可按独立仓库层追加历史成绩、趋势图、排行榜或多设备同步。
  - 若增加长期成绩记录，应先定义隐私边界、数据保留策略和导出/删除能力。

## 每日抉择
- 入口: `DailyDecisionToolPage`
- 页面编排: `lib/src/ui/pages/toolbox_daily_choice/daily_choice_hub.dart`
- 数据模型: `daily_choice_models.dart`
- 种子数据与指南: `daily_choice_seed_data.dart`
- `cook` 数据读取与缓存: `daily_choice_cook_service.dart`
- 远端菜谱库: S3 `/cook_data/daily_choice_recipe_library.db`（首次点击后下载到本地 `toolbox_daily_choice_recipes.db`，运行时只读本机 SQLite，不再随安装包打入 `recipe_library.json`）
- 外部导出资源: `D:\vocabularySleep-resources\cook_data`（同步生成 full JSON、summary manifest 与标准 SQLite 表结构，便于后续远端资源库分发）
- 本地自定义存储: `daily_choice_storage.dart`
- 第一版能力:
  - `吃什么`: 首次进入会提示点击加载菜谱库，并从 S3 `/cook_data` 下载标准 SQLite 到本地后使用；页面常驻内存只保留摘要与筛选索引，支持按餐段、厨具和个人食谱集开始随机、停止后锁定当前菜品，详情页再按菜谱 ID 读取结构化详细介绍、食材、完整步骤和关键提示；“做菜指南”已统一整合 `cook` 的“做菜之前”、本地基础技能和参考书目。
  - `穿什么`: 按气温档位和场景筛选随机；基于 `D:\vocabularySleep-resources\穿什么` 中多本本地 EPUB 资料整理出基础款、版型、TPO、色彩、材质、鞋配件与衣橱整理规则，并落地为 `87` 套温度/场景种子搭配、模块化穿搭指南、结构化详情说明和引导式衣橱管理。
  - `去哪儿`: 按 `出门 / 周边 / 远行` 与 `饮食 / 娱乐 / 运动 / 文化 / 历史 / 自然 / 学习 / 购物 / 社交 / 亲子 / 夜生活 / 放松 / 出片 / 特色区域 / 纪念` 双维筛选随机；当前内置 360 条常见场所 archetype 地点，详情页提供结构化出行说明、地图搜索词复制与 OpenStreetMap 检索链接，后续系统地图和开放地理数据接入仍独立推进。
  - `干什么`: 按运动、学习、出行、整理、放松、创作、社交随机，也支持“随机方向”。
  - `随机助手`: 支持临时录入自定义选项，在均匀、加权、联合分布多轮之间切换，并提供大转盘、骰子和硬币动画；骰子按 3 到 12 面拆分多骰子，硬币限制为两面均匀。
  - `决策助手`: 提供均匀随机、期望加权、因子评分、联合概率四种透明计算方式。
- `吃什么` 当前额外能力:
  - 午餐和晚餐允许重叠，同一菜品不再被强制切进单一餐段。
  - 高级设置支持“已有材料优先”、做法类型、荤素结构、清真/素食友好和常见忌口过敏等启发式筛选。
  - “已有材料优先”已改为 `exact -> strong -> broad` 三阶段随机池，先取多食材交集，再按高重叠与至少命中 1 项的相关菜谱扩池，避免结果长期只剩 1 道菜。
  - 多食材与自定义忌口均支持 chip 化添加/删除；常见忌口已补齐牛奶、花生、香菜、鱼腥草等入口，自定义项可继续补充蒜、姜、葱等个人雷区；紧凑短语如“番茄鸡蛋豆腐汤”也会拆出多个规范食材 token。
  - 支持个人食谱集：用户可创建集合，把内置菜、个人调整或自定义菜加入集合，并在集合范围内继续使用餐段、厨具、高级筛选和随机。
  - 管理页支持按菜名搜索、餐段/厨具/标签/食谱集过滤，并收口为折叠 + 分页式移动端布局；内置菜支持“不喜欢”、个人调整、另存为个人食谱、恢复原味，自定义编辑器支持新增、编辑、删除个人食谱并自动推断筛选属性。
  - “做菜指南”已整理为 `做菜之前 / 基础技能 / 食材匹配与筛选说明 / 参考书目` 等模块，统一承接 `cook` 基础信息、洗菜去残留、烘焙先称量等技能整理，并移除菜谱详情页中的逐条来源说明。
  - 详情页不再为吃什么逐条展示来源说明，统一在指南“参考书目”模块中呈现资料来源；《食物辑要》因竖排古体识别质量不稳，本轮未接入“食疗与禁忌”子页。
  - 页面性能已做专项收口：运行期不再常驻 7000+ 条 full JSON asset，菜谱筛选改为 `DailyChoiceEatCatalog` 预建索引，进入页面只读本地 SQLite 摘要与索引，完整详情按 ID 懒加载，管理页内置菜谱按页构建，远端库首次安装后长期复用本地缓存。
- `穿什么` 当前额外能力:
  - 默认读取 `AppState.weatherSnapshot`，按体感温度自动选中建议档位。
  - 若当前天气存在降水，会给出“切到雨天场景”的快捷建议，但不强制覆盖用户选择。
  - 若当前场景精确条目过少，会混入同温度稳妥备选，保证随机结果仍有变化。
  - 自定义编辑器支持勾选 `风格 / 版型 / 样式类型 / 面料与触感 / 亮点` 五组特征；管理页支持按 `风格 / 版型 / 样式类型` 过滤个人衣橱和内置搭配。
  - 指南入口已升级为 8 个模块：基础与风格、版型比例、场合职场、色彩材质、季节天气、鞋履配饰、衣橱练习、扩展边界。
- 可编辑边界:
  - 删除内置项只写入隐藏列表，不改种子数据。
  - 新增/编辑/删除自定义项和吃什么个人食谱集保存到应用支持目录下的 `toolbox_daily_choice_v1.json`。
  - `吃什么` 的主菜谱库会在首次点击后从 S3 `/cook_data` 下载到独立 SQLite `toolbox_daily_choice_recipes.db`；后续进入页面只读摘要和索引，完整菜谱在查看详情时按 ID 懒加载。
  - 吃什么内置菜的“个人调整”会保存在本地 `toolbox_daily_choice_v1.json` 中，优先级为“隐藏 > 个人调整 > 原始内置”；“另存为个人食谱”会生成独立自定义菜，不覆盖原内置条目。
  - 暂不侵入主数据库或 AppState，后续如需同步、备份和导入导出，可迁移到独立仓库层。
- 后续扩展预留:
  - 吃什么: 个人食谱集导入导出、跨端同步、可选手动刷新、成本/耗时等更细粒度筛选，以及仅在古籍识别质量稳定时再接入“食疗与禁忌”子页。
  - 穿什么: AI 真实数字人试穿、衣橱识别、购物网站比价和推荐。
  - 去哪儿: 基于粗略定位的动态半径检索、开放地图 POI、系统地图拉起、路线与营业状态联动。
  - 决策助手: 贝叶斯更新、回归/相关因子建模、多目标权重优化。

## 风险与后续路线
- 风险: 部分历史页面仍可通过独立入口被外部调用，需逐步覆盖统一守卫。
- 风险: 每日抉择第一版使用本地 JSON 存储，若用户清理应用数据，自定义项会一并清除。
- 风险: 吃什么个人食谱集当前仍基于本地 JSON 保存，尚未接入账号同步、备份导入导出或云端冲突合并。
- 风险: 管理页分页只限制 UI 一次性构建数量；若菜谱库扩大到数十万级，管理搜索应迁移到 SQLite 索引查询。
- 风险: 随机助手仅适合低风险、可回退、差异不大的选择，不应替代高风险决策分析或专业意见。
- 风险: 决策助手仅做透明计算和辅助排序，不适合作为医疗、法律、财务等高风险决策依据。
- 风险: 人类测试结果只适合作为趣味反馈，不能作为医学、心理、职业能力或教育评价依据。
- 路线:
  1. 将 remaining toolbox 子页面补齐模块 ID 与访问守卫。
  2. 清理遗留重复页面文件（例如历史 copy 版本）并收敛共享组件。
  3. 为每日抉择补充完整数据同步、导入导出和更细粒度的数学模型。
