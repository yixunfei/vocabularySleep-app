# 计划 047: 创意沙盘连续音效与移动端 UX 优化

## 基本信息
- **创建日期**: 2026-04-24
- **状态**: 已完成
- **负责人**: Codex

## 目标
修复创意沙盘滑动音效“只有一声、不持续、卡顿和空白明显”的问题，使连续滑动时以不中断的沙沙底噪为主，并在手机屏幕较小时优化首屏信息、画布空间和底部控制区。

## 详细步骤
1. 复查现有 `PLAN_044` / `PLAN_046` 相关改动，确认当前脏工作区中的音效服务和页面布局状态，不回退已有用户改动。
2. 修复 `toolbox_zen_sand_sound_service.dart` 中循环底噪生成与播放链路缺口：
   - 补齐当前已被调用但缺失的现代循环 PCM 生成函数。
   - 确保循环波形无首尾长静音、无明显拼接空白。
   - 保持连续型工具以 loop 底噪为主，避免滑动过程中高频短音反复切源。
3. 优化 `toolbox_zen_sand_tool.dart` 的小屏 UX/UI：
   - 375dp 左右屏宽优先保留画布和主操作。
   - 收紧顶部说明、统计信息和折叠底栏占用。
   - 保证底部控制区最小触控面积与可读性。
4. 补充或调整回归测试，覆盖循环音频的持续可听、首尾空白和波形动态稳定性。
5. 更新 `changelogs/CHANGELOG.md`，执行格式化、分析与测试验证。

## 风险评估
- **风险 1**: 当前工作区已存在同文件未提交改动，直接重写会覆盖已有修改。
- **缓解措施**: 仅基于当前文件增量修复缺失函数与布局细节，不做大范围重排。

- **风险 2**: 音频连续播放涉及平台播放器行为，单元测试无法完全模拟真机听感。
- **缓解措施**: 用纯 Dart 波形回归测试锁定“无空白”和“动态变化稳定”，运行时逻辑尽量减少切源和重启。

- **风险 3**: 小屏 UI 收紧可能导致部分辅助入口隐藏过深。
- **缓解措施**: 将高频操作保留在底部热区，把场景、工具、撤销、重做作为折叠态核心入口。
