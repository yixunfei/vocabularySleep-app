# 计划 017: Windows 本地 TTS 混合字段播放修复

## 基本信息
- **创建日期**: 2026-04-13
- **状态**: 已完成
- **负责人**: Codex

## 目标
1. 严格排查学习播放中“看似只播单词本体”的实际根因，避免继续围绕错误方向修补。
2. 修复 Windows 本地 TTS 在 `language = auto` 且未手动指定音色时，无法随字段内容切换合适语种/音色的问题。
3. 补充回归测试，验证本地 TTS 可连续处理多段播报，并对英文词头与中文释义做出不同语音选择。

## 详细步骤
1. 复核播放服务、TTS 服务与 Windows 插件实现，确认是否存在“只调用一次 speak”或“后续调用被 stop 截断”的问题。
2. 在 `lib/src/services/tts_service.dart` 中为 Windows 本地 TTS 增加可缓存的音色解析与按文本内容自动匹配逻辑。
3. 保持用户显式选择的本地音色优先级，只有在未指定音色时才根据字段内容自动匹配语言/音色。
4. 新增针对 Windows 本地 TTS 的回归测试，覆盖英文后接中文的连续播报场景。
5. 完成格式化、测试验证，并同步更新 `changelogs/CHANGELOG.md` 与本计划执行结果。

## 风险评估
- **风险 1**: Windows 平台 `flutter_tts` 的 `setVoice` 参数要求与其他平台不同，贸然复用旧逻辑可能继续失效。
- **缓解措施**: 仅在 Windows 分支中使用带 `locale + name` 的音色匹配，并保留语言级别回退路径。
- **风险 2**: 自动切换音色可能覆盖用户手动选择的声音偏好。
- **缓解措施**: 明确保持 `localVoice` 显式配置优先，仅在空音色配置下启用自动匹配。
- **风险 3**: 增加本地 TTS 预处理后，可能引入新的等待或超时问题。
- **缓解措施**: 通过测试覆盖连续两次 `speak` 的完成链路，并保持原有 completion/timeout 机制不变。

## 依赖项
- 本地依赖 `third_party/flutter_tts` 继续作为 Windows 插件实现来源。
- 学习播放链路维持现有 `PlaybackService -> TtsService` 调用结构。

## 执行结果
- 已确认学习播放主链路与 `PlaybackService` 会连续发起多个 `speak` 调用，当前问题不属于“只调用一次后循环提前停止”。
- 已为 Windows 本地 TTS 增加按文本内容自动匹配 `locale + name` 的音色选择逻辑，在未手动指定音色时可在英文词头与中文释义之间切换合适语音。
- 已为 Windows 本地 TTS 增加失败回退路径：当 `setVoice` 返回失败时，自动回退到 `setLanguage`，避免后续字段静默沿用错误声线。
- 已新增回归测试，覆盖英文后接中文的连续本地播报、显式音色优先级，以及 `setVoice` 失败后的语言回退场景。
