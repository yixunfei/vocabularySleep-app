# 计划 007: toolbox UI 共享样式收敛与结构整理

## 基本信息
- **创建日期**: 2026-04-09
- **状态**: 进行中
- **负责人**: Codex

## 目标
基于当前 `CHANGELOG` 中已完成的工具箱 UI 重建进度，以及 `docs/toolbox_design/TOOLBOX_DESIGN_REVIEW.md`、`docs/toolbox_design/TOOLBOX_ANIMATION_SPEC.md` 的设计反馈，对工具箱相关页面进行一轮更谨慎的 UI 收敛：在不改动业务逻辑、状态流、音频行为和动画时序的前提下，进一步统一共享样式、降低展示层重复代码，并改善文件结构与可维护性。

## 范围边界
- **包含**：
  - `toolbox` 入口页与通用页面壳的共享样式令牌收敛
  - `疗愈音钵`、`呼吸训练`、`静心念珠` 中纯展示层组件的样式复用与结构整理
  - 与本轮 UI 收敛直接相关的命名、拆分、注释和文件组织优化
- **不包含**：
  - 业务逻辑、状态持久化、手势规则、计时规则、音频播放逻辑、动画节奏参数
  - 文案语义变更、导航链路调整、功能行为变更
  - 跨 `toolbox` 范围的大规模全局主题迁移

## 详细步骤
1. 对照 `CHANGELOG` 中已完成的多轮 `toolbox` UI 精修项，识别仍分散在页面内部的展示层重复样式。
2. 从 `docs/toolbox_design` 中提炼可继续落地、且不会触碰逻辑层的设计反馈，聚焦卡片层次、按压反馈、面板边界、共享 chip 和按钮表达。
3. 在 `lib/src/ui/pages/toolbox/` 下补充共享 UI 基础设施：
   - 统一面板、胶囊按钮、图标按钮、信息 pill 等展示组件
   - 收敛尺寸、圆角、阴影、边框透明度等样式令牌
4. 将 `疗愈音钵`、`呼吸训练`、`静心念珠` 中可安全替换的纯 UI 容器接入共享展示层组件，保持入参、事件回调与状态来源不变。
5. 对受影响文件执行格式化、静态验证并更新 `CHANGELOG`，明确本轮仍然只涉及 UI 与结构维护。

## 风险评估
- **风险 1**: 共享组件抽取过度，导致页面原有视觉差异被抹平
- **缓解措施**: 只抽取布局骨架、卡片语言和通用交互容器，保留各工具页自己的色板与主题细节

- **风险 2**: 展示层重组时误触逻辑分支或状态绑定
- **缓解措施**: 不改变状态字段、回调签名和条件判断，仅替换包裹层与样式组件

- **风险 3**: 页面文件拆分后引用关系变复杂
- **缓解措施**: 优先在现有 `toolbox/` 目录中集中放置共享 UI 文件，避免跨层级散落
