# 计划 009: toolbox design 反馈闭环修复

## 基本信息
- **创建日期**: 2026-04-10
- **状态**: 进行中
- **负责人**: Codex

## 目标
重新梳理 `docs/toolbox_design/TOOLBOX_DESIGN_REVIEW.md` 与 `docs/toolbox_design/TOOLBOX_ANIMATION_SPEC.md` 中的设计反馈，对照当前 `changelogs/CHANGELOG.md` 与代码现状，补齐仍未闭环的工具箱设计问题；在可控范围内完成修复，并把结果同步记录到变更日志。

## 范围边界
- **包含**：
  - 统一缓动/时长规范的补完与接入
  - 工具箱色彩令牌抽取与入口页接入
  - `疗愈音钵` 中仍未完全对齐设计反馈的动效参数修复
  - `禅意沙盘` 双指手势冲突的缓解与轻量提示
  - `舒缓音乐` Blob 背景自然度的进一步修复
  - 与本轮反馈闭环直接相关的 `CHANGELOG` 更新
- **不包含**：
  - 与设计反馈无直接关系的业务功能扩展
  - 跨整个应用的大规模主题系统重构
  - 超出工具箱范围的架构迁移

## 详细步骤
1. 逐项对照 `toolbox_design` 文档中的问题清单、优先级建议与实现路线图，标记当前 `CHANGELOG` 已覆盖与未覆盖项。
2. 对未闭环的问题按风险和收益排序，优先修复：
   - 缓动/时长规范不足
   - 工具箱色彩令牌缺失
   - 疗愈音钵敲击与环境脉动参数未完全对齐
   - 禅意沙盘双指误触取消笔画
   - 舒缓音乐 Blob 仍偏机械
3. 在不破坏现有功能逻辑的前提下完成代码修复，并对受影响文件进行格式化与静态验证。
4. 更新 `CHANGELOG`，明确说明哪些设计反馈已被闭环修复，哪些仍属于后续规划项。

## 风险评估
- **风险 1**: 为修复交互反馈而误改业务逻辑或状态流
- **缓解措施**: 优先采用参数调整、展示层增强和轻量状态补充，不重写核心业务链路

- **风险 2**: 手势冲突修复引入新的绘制/缩放边界问题
- **缓解措施**: 使用最小必要的缓冲和提示机制，保持原有手势主流程不变

- **风险 3**: 动效优化增加重绘开销
- **缓解措施**: 复用现有动画控制器与 `RepaintBoundary`，避免引入每帧 `setState` 的新路径
