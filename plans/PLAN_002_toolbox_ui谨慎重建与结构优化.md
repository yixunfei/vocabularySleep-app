# 计划 002: toolbox UI 谨慎重建与结构优化

## 基本信息
- **创建日期**: 2026-04-09
- **状态**: 进行中
- **负责人**: Codex

## 目标
基于 `docs/toolbox_design/TOOLBOX_DESIGN_REVIEW.md` 与 `docs/toolbox_design/TOOLBOX_ANIMATION_SPEC.md` 中针对工具箱模块的设计反馈，对工具箱入口 UI 进行一轮谨慎重建；同时在不修改任何业务逻辑与交互逻辑的前提下，优化相关页面的结构组织、样式复用和代码可维护性。

## 范围边界
- **包含**：
  - `toolbox` 入口页的视觉层次、卡片样式、页面信息组织与响应式布局优化
  - `toolbox` 相关通用 UI 壳与样式常量的整理
  - 与本轮改造直接相关的命名、拆分、注释和文件结构优化
- **不包含**：
  - 各具体工具页内部的业务逻辑、状态逻辑、音频逻辑、动画节奏逻辑
  - 各工具页功能行为变更
  - 非 `toolbox` 直接相关模块的大范围样式迁移

## 详细步骤
1. 提炼 `toolbox_design` 文档中与入口页和通用交互相关的 P0/P1 反馈，形成可执行的视觉与结构调整清单。
2. 审查当前 `toolbox_page.dart` 和 `toolbox_tool_shell.dart` 的页面职责、组件粒度与样式重复点，标记可拆分区域。
3. 重构工具箱入口页：
   - 保持原有分组与导航逻辑不变
   - 引入更克制的卡片圆角、渐变层次、按压反馈和移动端友好的内容宽度控制
   - 将入口数据、分组布局、卡片组件拆分为更清晰的文件结构
4. 整理 `toolbox` 通用样式与页面壳：
   - 抽取可复用的尺寸、圆角、阴影、间距与内容宽度常量
   - 统一部分命名与结构，降低后续新增工具入口时的维护成本
5. 对修改文件进行格式化与静态验证，随后更新 `CHANGELOG` 记录本轮 UI/结构调整。

## 风险评估
- **风险 1**: UI 重构时误伤导航或入口映射
- **缓解措施**: 保持入口数据和 `pageBuilder` 绑定不变，只重组展示层代码

- **风险 2**: 样式抽取过度，导致工具页壳与现有视觉体系不协调
- **缓解措施**: 仅抽取 `toolbox` 范围内可直接复用的常量，不做跨全局主题的激进迁移

- **风险 3**: 为了“现代感”过度加入装饰，反而影响移动端可读性
- **缓解措施**: 遵循移动端优先和最大内容宽度 600dp 的约束，保持信息密度克制
