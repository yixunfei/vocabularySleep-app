## 核心原则

### 优先级排序
1. **移动端优先**: 手机屏幕空间不足时，优先使用抽屉(Drawer)折叠和子页面
2. **突出主题**: 保留并突出最重要的主题内容
3. **稳健性**: 代码必须健壮，模块必须可维护
4. **文档先行**: 修改前先创建计划，修改时对照计划，完成后先更新changelog，再对照当前进度
5. **模块化低耦合**：对于综合性应用尽可能降低各模块间的耦合程度，非必要不进行强侵入式编码（除非模块间本身存在一定内聚关联，或本身作为抽象服务层）

### 风险意识
- 识别潜在风险并文档化
- 评估每项决策的利弊
- 对歧义代码添加详细注释

## 移动端体验规范

### 屏幕空间管理

- 主题舞台信息尽可能在一屏内展示，可以左右折叠
- 必要时进行页面子级拆分避免长页面
- 注重移动手机用户操作习惯与体验
- 尽可能避免手机中水平滑动，如果超出屏幕考虑换行，缩写展开，或抽屉展示
- 注意避免层级关系与冲突（如“主舞台”内上下操作与页面上下滚动操作冲突被覆盖等低级错误）


### 响应式设计
- 基准设计: 375dp (iPhone SE)
- 最大内容宽度: 600dp
- 关键操作按钮: 最小触摸区域 48dp

## 编码与本地化

### 编码规范
```yaml
文件编码: UTF-8 (强制)
换行符: LF (Unix风格)
缩进: 2空格
命名: snake_case (变量/函数), PascalCase (类/组件)
```

### 国际化 (i18n)

所有用户可见文本必须使用 统一的国际化管理，拒绝写死的硬编码

## 文档管理

### 文档目录结构
```
├── AGENTS.md              # 本文件 - 代理开发指南
|── docs/
├── PROJECT_DOMAIN.md      # 项目整体说明文档
├── plans/                 # 计划文档目录
│   ├── PLAN_001_xxx.md    # 递增编号计划
│   └── PLAN_002_xxx.md
├── records/               #已经验收完结的归档
│   ├── record_001_xxx.md  # 递增编号计划
├── changelogs/            # 变更日志目录
│   └── CHANGELOG.md       # 主变更日志
├── modules/               # 模块专属文档
│   ├── module_01/         # 不同模块文档1
│   ├── module_02/         # 不同模块文档2
│   ├── module_02/         # 不同模块文档3
└── decisions/             # 技术决策记录
    └── DECISION_001_xxx.md
```

### 文档更新规则
1. **PROJECT_DOMAIN.md**: 任何项目范围修改时必须同步检查
2. **引用链路检查**: 修改后验证相关文档的引用完整性
3. **版本同步**: 文档版本号与代码版本保持一致

## 代码规范

### 模块拆分原则
| 模块 | 职责 | 文件大小建议 |
|------|------|-------------|
| 页面 (Pages) | 页面布局和状态 | < 200行 |
| 组件 (Widgets) | 可复用UI组件 | < 100行 |
| 服务 (Services) | 业务逻辑 | < 300行 |
| 模型 (Models) | 数据结构 | < 150行 |
| 仓库 (Repositories) | 数据访问 | < 200行 |

### 禁止事项
- ❌ 单文件超过 500 行
- ❌ 大类/上帝类
- ❌ 未分类的通用工具类
- ❌ 硬编码字符串(用户可见文本)

### 注释规范
```dart
// ============ 风险标注示例 ============
// [风险] 选择此实现的原因：
// 1. 当前业务场景需要...
// 2. 替代方案X存在...问题
// 3. 如未来需求变更，需考虑...
// ======================================

// [歧义] 这里的命名解释：
// "xxx" 指的是...，而非...（因为...）

// [TODO] 后续优化点：
// 当...时，可考虑使用...替代
```

### 命名规范
```dart
// 页面: xxx_page.dart
// 组件: xxx_widget.dart, xxx_tile.dart, xxx_button.dart
// 模型: xxx_model.dart
// 枚举: xxx_type.dart, xxx_status.dart
// 常量: xxx_const.dart
```

## 资源优化

### 包体大小控制
| 资源类型 | 策略 | 配置方式 |
|----------|------|----------|
| 图片 | WebP格式, 按需加载 | `flutter: assets:` 分组 |
| 字体 | 按需加载, subset | `assets/fonts/` 分离 |
| 视频/音频 | 远程CDN | `cached_network_image` |
| 大文件 | S3按需加载 | `dio` + 本地缓存 |

### S3 按需加载决策矩阵
```
资源大小    │ 网络状态   │ 加载策略
───────────┼───────────┼─────────────────
< 100KB    │ 任意       │ 内嵌打包
100KB-1MB  │ WiFi       │ 预加载
100KB-1MB  │ 移动网络   │ 按需加载
> 1MB      │ 任意       │ 按需加载 + 本地缓存
```

### 推荐开源库

## 决策流程

### 模糊概念处理
```
用户提出模糊需求
    ↓
清晰条例化梳理
    ↓
评估反馈
├── 可能风险?
├── 更好方案?
├── 约束条件?
    ↓
形成文档
    ↓
用户确认
    ↓
制定方案
    ↓
编码实现
```

```markdown
## 备选方案
1. 方案A: ...
2. 方案B: ...

## 后果
- 正面: ...
- 负面: ...
```
